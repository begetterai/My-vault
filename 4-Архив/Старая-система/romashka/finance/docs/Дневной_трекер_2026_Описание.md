# Ромашка — Дневной трекер 2026: описание

**Файл Google Sheets:** [Ромашка — Дневной трекер 2026](https://docs.google.com/spreadsheets/d/1_KFsr5IRXMb9_5IJiuJOD8OD29b793La8TH5n7nIJE4/edit)
**Скрипты:** `[[scripts/rebuild_daily_tracker.py]]` | `[[scripts/update_daily_tracker.py]]`
**Cron:** `0 23 * * * python3 /home/user/My-vault/scripts/update_daily_tracker.py`

---

## Назначение

Ежедневная сводка по каждой точке: выручка, оплаты по каналам, инкассация, остатки кассы, расходы. Даёт понимание дня в разрезе «кассовая смена» — как накладная, которую кассир сдаёт в конце смены, только автоматически.

---

## Листы

| Лист | Назначение |
|------|-----------|
| **ЗБ** | Точка Лохути 11 — дневные данные Jan–Dec 2026 |
| **ОВИР** | Точка Турсунзода — дневные данные Jan–Dec 2026 |
| **Свод** | Автосумма ЗБ + ОВИР через формулы |
| **Расходы ЗБ** | Детализация расходов по сменам (пока пустой — ручное заполнение) |
| **Расходы ОВИР** | То же для ОВИР |
| **Месяцы** | Помесячная сводка выручки и % выполнения плана |

---

## Структура строк (ЗБ, ОВИР, Свод)

- **Строка 1:** заголовок с названием точки и планом
- **Строка 2:** шапка колонок
- **Строки 3–367:** данные за каждый день 2026 года (Jan 1 = строка 3, Dec 31 = строка 367)

---

## Колонки — детальное описание

### Блок 1: Идентификация дня
| Кол. | Название | Тип | Источник |
|------|---------|-----|---------|
| A | Дата | Дата (дд.мм.гггг) | Формула при создании |
| B | День | Текст (Пн–Вс) | Формула при создании |

### Блок 2: Выручка
| Кол. | Название | Тип | Источник |
|------|---------|-----|---------|
| C | Выручка (с) | Авто | **Poster API:** `dash.getAnalytics` → `counters.revenue` |

Выручка = сумма всех закрытых заказов за день с учётом скидок. Скидки (комп/за счёт заведения) уже вычтены внутри Poster.

### Блок 3: Оплаты по каналам
| Кол. | Название | Тип | Источник |
|------|---------|-----|---------|
| D | Наличные | Авто | **Poster API:** `finance.getCashShifts` → `amount_sell_cash ÷ 100` |
| E | Alif | Вручную | Управляющий вносит из терминала/приложения Alif |
| F | DC | Вручную | Управляющий вносит из терминала Dushanbe City |
| G | Карта | Вручную | Управляющий вносит из POS-терминала |
| H | Beeygor / Teztar | Вручную | Управляющий вносит из приложения |

> Почему E–H вручную: Poster API возвращает общую сумму безнала (`amount_sell_card`), но не разбивает по платёжным системам. Разбивка есть только в каждом терминале отдельно.

| Кол. | Название | Тип | Источник |
|------|---------|-----|---------|
| I | Итого оплат | Формула | `=D+E+F+G+H` |
| J | Расхождение | Формула | `=C−I` (ноль = сходится; ненулевое = ошибка) |

Если J ≠ 0 — ячейка красная. Нужно проверить, не пропущен ли канал оплаты.

### Блок 4: Кассовые остатки и инкассация
| Кол. | Название | Тип | Источник |
|------|---------|-----|---------|
| K | Инкасс. нал. | Авто | **Poster API:** `finance.getCashShifts` → `amount_collection ÷ 100` (сумма инкассаций за смену) |
| L | Ост. откр. | Авто | **Poster API:** `finance.getCashShifts` → первая смена дня `amount_start ÷ 100` |
| M | Расходы | Авто | **Poster API:** `finance.getTransactions` → сумма транзакций `type=2` за день ÷ 100 |
| N | Ост. закр. | Авто | **Poster API:** `finance.getCashShifts` → последняя смена дня `amount_end ÷ 100` |

**Расходы (M)** — все финансовые расходные операции, проведённые в Poster за день: закупки, хозтовары, мелкие траты. Сумма всех категорий `type=2`. Детализация — в листе «Расходы ЗБ/ОВИР» или через скрипт `check_pnl_transactions.py`.

### Блок 5: KPI и план
| Кол. | Название | Тип | Формула |
|------|---------|-----|---------|
| O | Нарастающий итог | Формула | Накопительная сумма выручки с начала месяца; сбрасывается 1-го числа |
| P | % плана | Формула | `O ÷ план_месяца` |
| Q | Норма/день | Формула | `план ÷ количество_дней_в_месяце` |
| R | Откл. от нормы | Формула | `C − Q` (зелёный = выше нормы, красный = ниже) |
| S | Прогноз | Формула | `O ÷ день_месяца × дней_в_месяце` — экстраполяция по темпу |
| T | Нужно/день | Формула | `(план − O) ÷ оставшиеся_дни` — сколько нужно брать ежедневно |

**Планы:**
- ЗБ: 300 000 с/мес
- ОВИР: 360 000 с/мес
- Свод: 660 000 с/мес

## Лист Месяцы

Помесячная сводка: последняя строка каждого месяца из колонки O (нарастающий итог).

| Кол. | Содержание |
|------|-----------|
| A | Название месяца |
| B | План ЗБ (300 000 с) |
| C | Выручка ЗБ — ссылка на O{последний день месяца} листа ЗБ |
| D | % ЗБ = C ÷ B |
| E | План ОВИР (360 000 с) |
| F | Выручка ОВИР — ссылка на O{последний день} листа ОВИР |
| G | % ОВИР = F ÷ E |
| H | СВОД = C + F |
| I | % СВОД = H ÷ 660 000 |

## Автоматизация

**Cron** запускает `update_daily_tracker.py` ежедневно в 23:00 по серверному времени.

Скрипт для каждой точки (ЗБ, ОВИР):
1. Запрашивает 3 endpoint Poster с токеном точки
2. Определяет номер строки по формуле `строка = 3 + (дата − 01.01.2026).days`
3. Записывает данные в колонки C, D, K, L, M, N через `values:batchUpdate`

**Backfill** исторических данных — скрипт `rebuild_daily_tracker.py` (только для пересоздания всего файла).

## Токены Poster API

| Точка | Токен |
|-------|-------|
| ЗБ (Лохути 11) | `398711:8746917c4a23ea897774040e039dfb76` |
| ОВИР (Турсунзода) | `935215:79675564e3d086d7e03d5fd56b50c8df` |

> ⚠️ ОВИР: счёт `INV-039405` — оплатить до **01.05.2026**, иначе токен ОВИР заблокируют.

## Что нельзя автоматизировать из Poster

- **Разбивка безнала** (Alif / DC / Карта / Beeygor) — Poster даёт только общую сумму `amount_sell_card`, без детализации по терминалу
- **Поступления от поставщиков** и прочие приходные операции — управляются вне Poster
- **Банковские остатки** (Alif Pay, DC счёт) — нет API у этих систем
