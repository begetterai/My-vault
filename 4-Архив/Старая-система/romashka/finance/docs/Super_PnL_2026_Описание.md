# Ромашка — Super P&L 2026: описание

**Файл Google Sheets:** [Ромашка — Super P&L 2026](https://docs.google.com/spreadsheets/d/1l8Lau8K9997pyqJj-zjlILAkLoOHlQmPo6BAwqc5FBU/edit)

## Назначение

Управленческий P&L по формату: выручка → себестоимость → валовая прибыль → ФОТ → OpEx → EBITDA → чистая прибыль. Помесячно, с YTD-итогом. Отдельные листы на каждую точку + Свод + KPI-дашборд.

---

## Листы

| Лист | Содержание |
|------|-----------|
| **ЗБ** | P&L точки Лохути 11 |
| **ОВИР** | P&L точки Турсунзода |
| **Свод** | Суммарный P&L сети — формулы ссылаются на ЗБ и ОВИР |
| **KPI** | Дашборд ключевых метрик за март 2026 (последний полный месяц) |

---

## Структура строк (ЗБ, ОВИР, Свод)

- **Строка 1:** Заголовок с названием точки
- **Строка 2:** Подзаголовок (пометка о периоде заполнения)
- **Строка 3:** Пустая
- **Строка 4:** Шапка: Показатель | Янв | Фев | Мар | ... | Дек | YTD | Бюджет

Далее — блоки P&L, каждый начинается с заголовочной строки (синий фон).

---

## Блоки P&L и строки

### 📊 ВЫРУЧКА
| Строка | Показатель | Источник |
|--------|-----------|---------|
| Чистая выручка (Poster) | Продажи с учётом скидок | **Poster API** / ручной ввод |
| └ в т.ч. доставка | Доля доставки в выручке | Ручной ввод |
| Рабочих дней | Количество дней работы | Ручной ввод |
| Выручка в день (с) | Расчёт: выручка ÷ дни | Формула |

### 💳 ОПЛАТЫ
| Строка | Показатель | Источник |
|--------|-----------|---------|
| Наличные (с) | Сумма нала за месяц | **Poster API** `getCashShifts` / ручной ввод |
| Безналичные (с) | Итого безнал | Ручной ввод |
| └ Alif | Детализация | Ручной ввод |
| └ Dushanbe City | Детализация | Ручной ввод |
| └ Банковская карта | Детализация | Ручной ввод |
| % наличных | Нал ÷ (нал + безнал) | Формула |

### 🛒 СЕБЕСТОИМОСТЬ
| Строка | Показатель | Источник |
|--------|-----------|---------|
| Закупки кухня (с) | Стоимость продуктов | По данным поставок / Poster склад |
| Закупки бар (с) | Стоимость бара | По данным поставок / Poster склад |
| ИТОГО COGS | Кухня + Бар | Формула |
| Food Cost кухня % | COGS кухня ÷ выручка | Формула |
| Food Cost общий % | COGS итого ÷ выручка | Формула |

**Цель Food Cost: < 30%**

### ВАЛОВАЯ ПРИБЫЛЬ
| Строка | Показатель |
|--------|-----------|
| Валовая прибыль | Выручка − COGS |
| Маржа % | Валовая прибыль ÷ выручка |

**Цель маржи: > 60%**

### 👥 ФОТ
| Строка | Показатель | Источник |
|--------|-----------|---------|
| ФОТ — производство | Зарплаты поваров, кассиров | Ведомость / Poster |
| ФОТ — администрация | Зарплаты управляющих, АХЧ | Ведомость |
| ИТОГО ФОТ | Сумма | Формула |
| Labor Cost % | ФОТ ÷ выручка | Формула |
| Prime Cost % | (COGS + ФОТ) ÷ выручка | Формула |

**Цель Labor Cost: < 25% | Prime Cost: < 55%**

### 📋 ОПЕРАЦИОННЫЕ РАСХОДЫ (OpEx)
| Строка | Показатель | Источник |
|--------|-----------|---------|
| Аренда | По договору | Ручной ввод |
| % аренды | Аренда ÷ выручка | Формула |
| Коммунальные | Электричество, вода, газ | Ручной ввод / счета |
| Упаковка | Контейнеры, стаканы | Poster склад / ручной |
| Логистика | Доставка продуктов | Ручной ввод |
| CRM / Poster | Подписка на систему | Фиксированная сумма |
| Хозтовары | Чистящие, расходники | Poster / ручной |
| Маркетинг | Реклама, промо | Ручной ввод |
| Юридические расходы | Юрист, оформление | Ручной ввод |
| Прочие расходы | Всё остальное | Ручной ввод |
| Форс-мажор | Непредвиденные | Ручной ввод |
| ИТОГО OpEx | Сумма всех выше | Формула |

### EBITDA
| Строка | Показатель |
|--------|-----------|
| EBITDA | Валовая прибыль − ФОТ − OpEx |
| EBITDA % | EBITDA ÷ выручка |

**Цель EBITDA: 15–22%**

### 💰 ИТОГ
| Строка | Показатель | Источник |
|--------|-----------|---------|
| Налоги | Налоговые платежи | Ручной ввод / налоговая |
| Чистая прибыль | EBITDA − налоги | Формула |
| Чистая маржа % | Чистая прибыль ÷ выручка | Формула |
| Дивиденды | Выплаты учредителям | Ручной ввод |
| Свободный кэш | Чистая прибыль − дивиденды | Формула |

**Цель чистой маржи: 10–15%**

### 📱 POSTER МЕТРИКИ
| Строка | Показатель | Источник |
|--------|-----------|---------|
| Транзакции (чеки) | Количество закрытых заказов | **Poster API** `dash.getAnalytics` |
| Гости | Количество гостей | **Poster API** `dash.getAnalytics` |
| Средний чек (с) | Выручка ÷ транзакции | Poster / формула |
| Гостей в день | Гости ÷ рабочих дней | Формула |

## Важно: транзакции в трекере vs P&L

В **дневном трекере** колонка M (Расходы) — это `finance.getTransactions type=2` из Poster. Туда попадают ВСЕ расходные операции дня: закупки, хозтовары, выплаты, аренда (если проведена через кассу). Это не то же самое, что строки OpEx в P&L.

В **Super P&L** каждая категория расходов — отдельная строка. Их нужно сопоставлять с категориями Poster:

| Poster категория | Строка P&L |
|-----------------|-----------|
| Поставки | Закупки кухня / бар |
| ФОТ Производственный | ФОТ — производство |
| ФОТ Административный | ФОТ — администрация |
| Аренда | Аренда |
| Налоги | Налоги |
| Коммунальные | Коммунальные |
| Хозтовары | Хозтовары |
| Упаковка | Упаковка |
| Выплаты дивидендов | Дивиденды |
| Прочие | Прочие расходы |

Для проверки: `[[scripts/check_pnl_transactions.py]]`
