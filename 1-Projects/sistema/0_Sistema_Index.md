## Архитектура синхронизации

**Принцип:** данные (суммы, даты) живут в Google Drive (Sheets), архитектура (структура, статусы, решения, задачи) — в Obsidian. Между двумя маркерами `<!-- DRIVE-SYNC:START -->` / `<!-- DRIVE-SYNC:END -->` в заметке — автоматически подтягиваемый снимок, остальное правится вручную.

- **Sheet:** [«Система — Подписки»](https://docs.google.com/spreadsheets/d/1tP1xPKU6BO3w9zbhru7dhAsk5nijDl_9Ii9tyF9SAYo) (папка Drive: Private → Трекеры)
- **Скрипт синка:** `scripts/sync_sistema.py` — читает Sheet, обновляет блок в [[1-Projects/sistema/подписки|подписки.md]], коммитит и пушит через `./cli.sh sync`
- **Запуск вручную:** `python3 scripts/sync_sistema.py`

### Автоматический запуск — статус и ограничение

Зарегистрирована cron-задача (`CronCreate`, job `00f9cec3`, ежедневно 09:13) с самопродлением (после каждого запуска она пересоздаёт себя, чтобы обойти 7-дневное автоистечение recurring-задач).

**Важная находка:** в этой облачной среде `CronCreate` явно подтвердил, что задача session-only — "dies when Claude exits" — даже с флагом `durable: true`. То есть пока этот контейнер/сессия жива, задача будет работать (с самопродлением — до недели и дальше). Но если сессия завершится или контейнер будет переработан после простоя, задача исчезнет вместе с ней — это не настоящий OS-level cron.

**Для настоящей бессрочной автоматизации** нужен Trigger на уровне среды (Claude Code on the web — конфигурируется в настройках среды, не из чата): https://code.claude.com/docs/en/claude-code-on-the-web. Это отдельный механизм, который поднимает новую сессию по расписанию независимо от того, жива ли текущая. Пока не настроено — если снимок данных в подписки.md давно не обновлялся (смотри дату "обновлено" в блоке), значит cron-цепочка прервалась и её нужно пересоздать.
